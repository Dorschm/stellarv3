# Map Generation

## Overview

Maps are generated from PNG files by the Go CLI in `map-generator/`. The pipeline:

```
PNG image (grayscale blue channel)
    → go run . (map-generator CLI)
    → manifest.json, map.bin, map4x.bin, map16x.bin, thumbnail.webp
    → copied into resources/maps/<mapname>/
```

## Running the map generator

```bash
# Generate all maps (runs Go, then formats)
npm run gen-maps

# Under the hood:
cd map-generator && go run .

# Generate only specific maps
cd map-generator && go run . --maps=sol_system,asteroid_belt

# Verbose output
cd map-generator && go run . --verbose

# Debug logging
cd map-generator && go run . --log-level=DEBUG
```

## Adding a new map

1. **Create a source PNG** in `map-generator/assets/maps/<mapname>/image.png`.

   - Size: 2–3 million pixels recommended (e.g., 2000×1000).
   - The **blue channel** encodes terrain type (see pixel → terrain table below).
   - Minimum island: 30 contiguous sector tiles.
   - Minimum "lake" (VoidPocket): 200 contiguous void tiles.

2. **Create `map-generator/assets/maps/<mapname>/info.json`**:

   ```json
   {
     "name": "Sol System",
     "mapName": "sol_system",
     "category": "space",
     "width": 2000,
     "height": 1000
   }
   ```

3. **Run the generator**:

   ```bash
   npm run gen-maps
   ```

   Output files appear in `resources/maps/<mapname>/`.

4. **Register the map** in `src/core/game/Game.ts`:
   ```typescript
   export enum GameMapType {
     SolSystem = "Sol System",
     // add here
   }
   export const mapCategories: Record<string, GameMapType[]> = {
     space: [..., GameMapType.SolSystem],
   };
   ```

## Pixel → Terrain type mapping (blue channel)

| Blue value range | Terrain type     | In-game name               |
| ---------------- | ---------------- | -------------------------- |
| 0                | DeepSpace (void) | Deep space between sectors |
| 1–9              | OpenSpace        | Habitable, low-cost tiles  |
| 10–19            | Nebula           | Intermediate terrain       |
| 20–255           | AsteroidField    | High-cost, resource-rich   |

The map generator reads the blue channel and encodes the magnitude into the binary format.
The `terrainType()` method in `GameMapImpl.ts` decodes it at runtime.

## Output file formats

| File             | Description                                   |
| ---------------- | --------------------------------------------- |
| `map.bin`        | Full-resolution binary terrain + metadata     |
| `map4x.bin`      | 4× downscaled (used for minimap rendering)    |
| `map16x.bin`     | 16× downscaled (used for sector topology)     |
| `thumbnail.webp` | Preview image for lobby map selection         |
| `manifest.json`  | Map metadata (dimensions, sector count, etc.) |

## `scripts/generate-space-maps.mjs`

This Node.js script (13.5 KB) post-processes maps after Go generation. It handles:

- Registering new maps in the production manifest
- Any JS-side transforms on the binary output

It's invoked as part of `npm run gen-maps` after the Go step.

## Test maps

Test maps (in `tests/testdata/maps/`) are **not** generated by this pipeline — they were
created manually as small binary files. To create a new test map:

1. Create a small PNG (e.g., 100×50) with the pixel encoding above.
2. Run the Go generator with `--maps=<mapname>`.
3. Copy the output to `tests/testdata/maps/<mapname>/`.
4. Use the map name in `setup("mapname", {...})` in tests.

Available test maps: `ocean_and_land`, `plains`, `big_plains`, `half_land_half_ocean`,
`giantworldmap`, `world`.

## Go CLI flags

```
--maps=<name1,name2>      Process only these maps (comma-separated directory names)
--log-level=DEBUG|INFO|WARN|ERROR|ALL
--verbose, -v             Enable debug logging
--debug-performance       Log timing per map step
--debug-removal           Log tile removal decisions
```
